# Astrobot Telegram: сценарий бота

Дата фиксации: 17 июня 2026.

## Цель документа

Описать первый рабочий пользовательский путь Telegram-бота: от входа в бота до покупки услуги и получения AI-разбора.

Этот документ является основой для разработки MVP.

## Главный принцип

Бот должен не выглядеть как каталог услуг. Он должен вести пользователя как мягкий персональный помощник:

1. Быстро дать бесплатную пользу.
2. Понять интерес пользователя.
3. Предложить подходящий платный разбор.
4. Собрать минимум нужных данных.
5. Принять оплату через Telegram Stars.
6. Сгенерировать результат.
7. Предложить следующий логичный шаг.

## MVP-сценарии

В первую версию входят:

1. Бесплатный дневной прогноз.
2. Платный личный вопрос.
3. Платная совместимость.
4. Платная натальная карта.
5. Платный прогноз на месяц.

Позже добавляются:

6. Нумерология.
7. Таро + астрология.
8. Лунный календарь.

## Главное меню

Кнопки главного меню:

- Мой прогноз

Расширенное меню после MVP:

## Первый вход в бота

### Команда `/start`

Сообщение:

Привет. Я Astrobot, твой персональный астрологический помощник.

Я могу сделать прогноз, разобрать совместимость, ответить на личный вопрос или собрать твой астропортрет по дате рождения.

Для начала я могу бесплатно подготовить короткий прогноз на сегодня.

Кнопки:

- Получить прогноз
- Посмотреть услуги
- Заполнить профиль

## Профиль пользователя

Профиль нужен, чтобы не спрашивать одни и те же данные перед каждой услугой.

### Данные профиля

Обязательные:

- Имя
- Дата рождения

Желательные:

- Время рождения
- Место рождения

Опциональные:

- Пол
- Часовой пояс

### Если пользователь не знает время рождения

Бот отвечает:

Ничего страшного. Я сделаю разбор без точного времени рождения. Некоторые детали будут менее точными, но общий портрет и основные тенденции все равно можно описать.

Кнопки:

- Продолжить без времени
- Указать примерное время
- Вернуться назад

## Бесплатный дневной прогноз

### Цель

Дать пользователю быстрый результат и привести к первой покупке.

### Путь

1. Пользователь нажимает "Получить прогноз".
2. Бот проверяет, есть ли имя и дата рождения.
3. Если данных нет, бот собирает имя и дату рождения.
4. Бот генерирует короткий дневной прогноз.
5. После результата предлагает расширенный прогноз или другую услугу.

### Данные

- Имя
- Дата рождения

### Результат

Короткий текст на 800-1200 знаков:

1. Главная энергия дня.
2. Любовь и эмоции.
3. Работа и деньги.
4. Совет дня.

### Апсейл после результата

Сообщение:

Если хочешь глубже, я могу сделать расширенный прогноз: любовь, работа, деньги, риски дня и лучшее действие.

Кнопки:

- Расширить прогноз, 100 Stars
- Прогноз на месяц
- Вернуться в меню

## Платный личный вопрос

### Цель

Самый простой платный продукт для первой покупки.

### Путь

1. Пользователь нажимает "Задать вопрос".
2. Бот предлагает выбрать сферу.
3. Пользователь выбирает сферу.
4. Бот просит написать один точный вопрос.
5. Бот показывает цену.
6. Пользователь оплачивает Stars.
7. Бот генерирует ответ.
8. Бот предлагает углубленную услугу по теме вопроса.

### Сферы

- Любовь
- Работа
- Деньги
- Саморазвитие
- Семья
- Другое

### Данные

Обязательные:

- Сфера
- Вопрос

Желательные:

- Имя
- Дата рождения

### Цена MVP

350 Stars.

### Результат

Текст на 2500-4000 знаков:

1. Короткий прямой ответ.
2. Что влияет на ситуацию.
3. Чего пользователь может не замечать.
4. Лучший следующий шаг.
5. Чего лучше избегать.
6. Поддерживающее завершение.

### Апсейл

- Если сфера "Любовь": предложить совместимость.
- Если сфера "Работа" или "Деньги": предложить натальную карту.
- Если вопрос о сроках: предложить прогноз на месяц.

## Совместимость

### Цель

Сильный продукт для любовной и эмоциональной аудитории.

### Путь

1. Пользователь нажимает "Совместимость".
2. Бот спрашивает тип отношений.
3. Бот берет данные пользователя из профиля или собирает их.
4. Бот собирает данные второго человека.
5. Бот показывает короткий тизер.
6. Бот предлагает полный разбор за Stars.
7. После оплаты генерирует совместимость.
8. После результата предлагает личный вопрос или любовный прогноз.

### Тип отношений

- Романтические
- Дружеские
- Деловые
- Семейные
- Пока не знаю

### Данные пользователя

- Имя
- Дата рождения
- Время рождения, если известно
- Место рождения, если известно

### Данные второго человека

- Имя или псевдоним
- Дата рождения
- Время рождения, если известно
- Место рождения, если известно

Кнопки:

- Вернуться в меню

## Оплата

### Основной способ MVP

Telegram Stars.

### Логика оплаты

1. Бот показывает описание услуги и цену.
2. Пользователь нажимает кнопку оплаты.
3. Бот отправляет invoice в Stars.
4. После успешной оплаты бот получает `successful_payment`.
5. Только после этого начинается генерация результата.
6. Заказ и платеж сохраняются в базе.

### Если пользователь не оплатил

Через 15 минут можно отправить мягкое напоминание:

Твой разбор почти готов к запуску. Если хочешь продолжить, оплата все еще доступна.

Кнопки:

- Оплатить
- Выбрать другую услугу
- Вернуться в меню

## Выдача результата

### Состояния

После оплаты:

Спасибо, оплата прошла. Я собираю твой разбор. Обычно это занимает до минуты.

Во время генерации:

Я уже анализирую данные и собираю ответ в понятную структуру.

После генерации:

Готово. Вот твой разбор.

### Если генерация не удалась

Сообщение:

Не получилось подготовить разбор с первого раза. Я уже пробую еще раз.

Если повторная попытка не удалась:

Сейчас возникла техническая ошибка. Оплата сохранена, заказ не потерян. Я сообщу, когда разбор будет готов.

Администратору отправляется уведомление.

Администратор получает уведомления:

- Новый пользователь
- Новый заказ
- Успешная оплата
- Ошибка генерации
- Запрос в поддержку
- Запрос возврата

## Что сохраняем в базе

### Пользователь

- Telegram ID
- Username
- Имя
- Дата рождения
- Время рождения
- Место рождения
- Дата регистрации

### Заказ

- ID заказа
- Telegram ID пользователя
- Тип услуги
- Статус заказа
- Цена в Stars
- Данные для услуги
- Результат AI
- Дата создания
- Дата оплаты
- Дата выдачи результата

### Платеж

- ID платежа
- ID заказа
- Telegram payment charge ID
- Цена в Stars
- Статус
- Дата оплаты
- Дата возврата, если был возврат

## Статусы заказа

- created — заказ создан
- waiting_payment — ожидает оплату
- paid — оплачен
- generating — идет генерация
- delivered — результат выдан
- failed — ошибка
- refunded — возврат

## Тон общения

Бот говорит:

- мягко;
- уверенно;
- без давления;
- без страшных предсказаний;
- без обещаний абсолютной точности;
- с практическим выводом.

Бот не говорит:

- "Это точно случится"
- "У вас нет выбора"
- "Этот человек ваша судьба"
- "Вам обязательно нужно расстаться"
- "Не ходите к врачу"
- "Вложите деньги сюда"
- "Я гарантирую результат"
