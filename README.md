# Astrobot Telegram

Первая тестовая версия Telegram-бота для автоматизированных астрологических услуг.

## Что уже работает

- команда `/start`;
- приветственный экран;
- главное меню;
- тестовый бесплатный прогноз;
- описание услуг;
- пошаговая анкета личного профиля;
- просмотр, изменение и удаление профиля;
- локальное хранение данных в SQLite;
- персональный дневной прогноз;
- персональный ответ на личный вопрос;
- разбор совместимости двух людей;
- сбор типа отношений и данных второго человека;
- натальная карта с кнопочным уточнением времени рождения и главного фокуса;
- прогноз на текущий или следующий месяц;
- выбор общего прогноза или главной сферы месяца;
- история последних заказов;
- повторный просмотр и удаление сохранённых результатов;
- выбор сферы, проверка вопроса и безопасные ограничения;
- сохранение тестовых заказов и результатов;
- демонстрационный и OpenAI-режимы генерации;
- сохранение одного прогноза на пользователя в день;
- платежи Stars, поддержка и защищённые админ-команды.

## Первый запуск

1. Открой файл `.env` и вставь токен после `BOT_TOKEN=` без кавычек.
2. Создай виртуальное окружение:

```powershell
python -m venv .venv
```

3. Установи зависимости:

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

4. Запусти бота:

```powershell
.\.venv\Scripts\python.exe -m bot.main
```

Для остановки нажми `Ctrl+C`.

После изменения списка зависимостей достаточно снова выполнить:

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

База `astrobot.db` создается автоматически при первом запуске.

## AI-режим

По умолчанию используется безопасный демонстрационный режим:

```text
AI_PROVIDER=demo
```

Чтобы включить OpenAI API:

1. Создай секретный ключ на `https://platform.openai.com/api-keys`.
2. В `.env` измени только эти строки:

```text
AI_PROVIDER=openai
AI_API_KEY=твой_секретный_ключ
AI_MODEL=gpt-4.1-mini
```

3. Перезапусти бота.

Не отправляй API-ключ в переписке и не публикуй файл `.env`. В OpenAI-режиме бот передает сервису данные профиля, необходимые для персонализации прогноза. Перед генерацией пользователь видит соответствующее уведомление.

## Telegram Stars

Режим оплаты задаётся в `.env`:

```text
PAYMENTS_MODE=demo
SUPPORT_USERNAME=
ADMIN_TELEGRAM_ID=
```

- `demo` — invoice не создаётся, услуги работают бесплатно;
- `stars_test` — создаётся настоящий invoice на 1 Star;
- `stars` — используются цены каталога: 350, 600, 900 и 400 Stars.

Для `stars` обязательно настроить `ADMIN_TELEGRAM_ID`, `SUPPORT_USERNAME`, `AI_PROVIDER=openai` и `AI_API_KEY`. `ADMIN_TELEGRAM_ID` — положительный числовой Telegram ID владельца бота. Для цифровых услуг бот использует валюту `XTR`, пустой provider token и запускает генерацию только после события `successful_payment`.

Команды поддержки: `/terms`, `/support`, `/paysupport`.

Админские команды доступны только пользователю из `ADMIN_TELEGRAM_ID`:

- `/admin` — справка;
- `/admin_orders` — последние заказы;
- `/admin_order <ID>` — подробности заказа;
- `/admin_user <Telegram ID>` — данные пользователя;
- `/admin_retry <ID>` — повторить генерацию оплаченного заказа после ошибки;
- `/admin_refund <ID>` — вернуть пользователю Stars через Telegram.

Бот уведомляет администратора о новых пользователях и заказах, успешной оплате,
ошибках платежа и генерации, а также обращениях в поддержку.

## Безопасность

Файл `.env` содержит секреты и уже добавлен в `.gitignore`. Не публикуй токен бота и не отправляй его в сообщения.
