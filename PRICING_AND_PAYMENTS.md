# Astrobot Telegram: ценовая сетка и оплаты

Дата фиксации: 17 июня 2026.

## Вывод

Для MVP выбираем простую модель:

1. Основная валюта внутри Telegram: Telegram Stars (`XTR`).
2. Витринные цены в тексте можно показывать в USD, GEL и RUB, но фактическая оплата внутри бота для цифровых услуг должна идти через Stars.
3. На старте не подключаем сложный эквайринг, пока не появится внешний сайт, юридическая структура и понятная страна приема платежей.
4. Для русскоязычной аудитории можно дополнительно показывать ориентиры в рублях, но принимать оплату лучше через Stars, чтобы не упираться в карты, банки, Apple/Google и страновые ограничения.

## Способы оплаты

### 1. Telegram Stars

Статус: главный способ для MVP.

Плюсы:

- Нативно внутри Telegram.
- Подходит для цифровых услуг.
- Пользователь не вводит карту боту.
- Работает на iOS, Android и desktop.
- Меньше проблем с правилами Apple/Google.
- Простая техническая интеграция через Bot API.

Минусы:

- Пользователю может быть непривычно покупать Stars.
- Вывод денег идет по правилам Telegram/Fragment, нужно отдельно проверить доступность вывода для нашего аккаунта и юрисдикции.
- Цена Stars для пользователя может отличаться из-за комиссий, НДС и страны.

Вывод: берем как основной вариант.

### 2. Банковские карты через Telegram Payments

Статус: не основной для цифровых услуг внутри Telegram.

Плюсы:

- Привычный способ оплаты.
- Можно подключить Stripe, YooKassa или другой провайдер, если есть подходящая страна и юридическое лицо.
- Telegram не берет комиссию за Payments API, комиссию берет платежный провайдер.

Минусы:

- Для цифровых услуг внутри Telegram Stars обязательнее и безопаснее с точки зрения правил платформ.
- Нужен платежный провайдер, KYC, договор, страна, налоговая логика.
- Возможны chargeback, споры, возвраты.

Вывод: подключать позже для внешнего сайта, консультаций, PDF вне Telegram или офлайн-услуг.

### 3. Stripe

Статус: хороший международный вариант, если есть доступная юрисдикция.

Плюсы:

- Сильная документация и API.
- Поддержка карт, Apple Pay, Google Pay.
- Удобен для международной аудитории.

Минусы:

- Работает не во всех странах.
- Нужна регистрация бизнеса и проверка.
- Есть комиссии и chargeback.

Вывод: отлично для внешнего сайта, но не главный способ внутри Telegram MVP.

### 4. YooKassa

Статус: вариант для РФ/рублевой аудитории, если есть подходящая регистрация.

Плюсы:

- Знакома российским пользователям.
- Поддерживает карты, ЮMoney, интернет-банкинг, счета в мессенджерах.
- Есть сценарии оплаты в Telegram.

Минусы:

- Зависит от российской юридической и банковской инфраструктуры.
- Не лучший вариант для международного запуска.
- Для цифровых услуг внутри Telegram все равно лучше Stars.

Вывод: рассмотреть позже, если основной рынок будет РФ.

### 5. CloudPayments / Robokassa / Prodamus и похожие агрегаторы

Статус: запасные варианты.

Плюсы:

- Могут быть удобны для СНГ/инфобизнеса.
- Часто дают платежные ссылки.
- Подходят для ручного MVP или внешней страницы оплаты.

Минусы:

- Не всегда идеально встраиваются в Telegram Bot Payments.
- Нужны договоры, комиссии, возвраты, фискализация.
- Для цифровых услуг внутри Telegram лучше не обходить Stars.

Вывод: использовать только если Stars не подойдет для вывода или если делаем внешнюю страницу оплаты.

### 6. Криптовалюта / TON / USDT

Статус: не для MVP.

Плюсы:

- Может быть удобно части международной Telegram-аудитории.
- Быстрые переводы.

Минусы:

- Внутри Telegram для цифровых услуг нельзя заменять Stars криптовалютой.
- Выше юридические, налоговые и пользовательские риски.
- Для массовой аудитории сложнее и менее доверительно.

Вывод: не используем на старте.

### 7. Ручная оплата переводом

Статус: только временная аварийная схема.

Плюсы:

- Можно стартовать без интеграции.
- Подходит для первых закрытых тестов.

Минусы:

- Нужно вручную проверять оплату.
- Плохая масштабируемость.
- Риск ошибок и споров.
- Низкое доверие у новых пользователей.

Вывод: можно использовать только для первых 10-20 тестовых клиентов, если платежная интеграция еще не готова.

## Самый беспроблемный стек оплат для запуска

1. MVP: Telegram Stars.
2. После первых продаж: добавить учет покупок, возвраты через `refundStarPayment`, команды `/terms`, `/support`, `/paysupport`.
3. После подтверждения спроса: внешний лендинг с оплатой картой через Stripe или YooKassa, в зависимости от страны бизнеса.
4. После масштабирования: подписки, пакеты, промокоды и партнерская программа.

## Что нужно сделать перед запуском оплат

1. Проверить доступность вывода Stars для нашего аккаунта и страны.
2. Подготовить условия использования.
3. Подготовить политику возвратов.
4. Добавить команду поддержки.
5. Добавить журнал заказов и платежей.
6. Сделать тестовые платежи.
7. Зафиксировать, что результат является развлекательным/саморефлексивным контентом, а не медицинской, юридической, финансовой или психологической консультацией.
